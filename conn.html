Conceptualizing GameScript (May 2025)

I first starting thinking of GameScript way back in May of 2025
It was during my school's summer holidays, I was bored and wanted to code a game
I was trying to code chess but with a pixel art style
I used to code games in p5.js in javascript itself.
It was only then I realised that javascript, in all it's glory, was interpreted which isn't a bad thing but it can cause unpredictable behaviour
Then I thought if there just could be a coding langauge which could be simple like javascript and python but also compiled like C++
This led me to think about how languages actually function and behave underneath the surface
After that discovery I realised I could try to make one for the vision I saw
Then I went on and talked to some friends of mine about the things they don't like in already existing languages
I thought about it and started to design a whole syntax of a language
oftentimes my sytax felt like just renaming stuff and looking back, that looks horrendous
Like 'display()' instead of 'print()' ? What was I thinking?
Later on I subtled my ideas and focused to make the language be accessible and easy and offer multiple syntaxes so that it's easy to adapt
I didn't yet code the ideas, just let them sit there in my mind and moved on
